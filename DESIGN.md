# WBS Manager 設計書

## 概要

| 項目 | 内容 |
|------|------|
| ツール名 | WBS Manager |
| バージョン | v1.29.0 |

## 機能一覧

### 1. タスク・フォルダ管理

#### 1.1 階層構造

| 機能 | 概要 |
|------|------|
| 自由階層 | タスクとフォルダを任意の深さで入れ子にできる。階層数の制限なし |
| フォルダ | タスクをグループ化する入れ物。進捗率は子孫タスクから自動計算（期間加重平均） |
| タスク | 実作業単位。進捗率は手動入力 |
| 折りたたみ | フォルダ単位で子要素を折りたたみ・展開できる（リスト・ガント共通） |

#### 1.2 タスク属性

| 属性 | 概要 |
|------|------|
| 名前 | タスク・フォルダの名称 |
| 成果物定義 | そのタスクの完了条件・成果物を記述 |
| ステータス | 未着手 / 進行中 / 完了 / ブロック中 の 4 段階 |
| 担当者 | `<input list>` + `<datalist>` による1ボックス入力（登録ユーザー候補＋自由入力） |
| 開始日 / 終了日 | YYYY-MM-DD 形式。子タスク追加時は親の日程を継承 |
| 進捗率 | 0〜100%（タスクのみ手動入力。フォルダは自動計算） |
| 依存関係 | 前工程タスクの ID を複数指定可能 |
| 備考 | メモ・補足情報 |
| 非表示フラグ | `hidden:true` でリスト・ガントから非表示化（データは保持） |

#### 1.3 CRUD 操作

| 操作 | 方法 |
|------|------|
| 追加 | ヘッダーボタン / 行左端のSVGアイコン / 右クリックメニュー |
| 編集 | 行クリックで右パネル表示、フィールド直接編集 |
| 削除 | 行左端のゴミ箱アイコン / 右クリックメニュー（子孫も同時削除） |
| 並替 | 「⠿ 並替モード」ON 後、行左端ハンドルをドラッグ＆ドロップ |
| 非表示 | 行左端の👁アイコンをクリック（個別トグル） |

### 2. リストビュー

| 機能 | 概要 |
|------|------|
| 階層インデント表示 | タスクの深さに応じてインデント表示。アイコンで種別（📁 / 📄）を識別 |
| 行番号（No） | 階層に応じた表示番号を動的計算（例：1, 1.1, 2.3.1） |
| 進捗バー | タスクは進捗率に応じたカラーバー。フォルダは自動計算値を表示 |
| 担当者アバター | イニシャルとユーザーカラーで視覚表示 |
| 期限ハイライト | 期限超過は赤文字、当日はオレンジで強調表示 |
| 依存関係表示 | 前工程の表示 No をチップ形式で表示 |
| フォルダ日程矛盾 | 子タスクの日程がフォルダ期間を超える場合に ⚠ マークと件数を表示 |
| 検索 | タスク名・成果物のテキスト検索 |
| ステータスフィルタ | 全ステータス / **完了以外** / 未着手 / 進行中 / 完了 / ブロック中 |
| 担当者フィルタ | 担当者名で絞り込み |
| 非表示機能 | 行左端👁アイコンで個別非表示。ツールバーのチェックボックスで非表示タスクを半透明表示・個別再表示可能 |
| ヘッダー固定 | `padding-top:0` によりスクロール時もテーブルヘッダーが最上端に固定表示 |
| 行アクションアイコン | 行最左端に固定配置（同列追加 / 子追加 / 非表示 / 削除のSVGアイコン）。並替モード時はドラッグハンドルのみ表示 |

### 3. ガントチャート

#### 3.1 表示スケール

| スケール | 列幅 | 上段ヘッダー | 下段ヘッダー |
|---------|------|------------|------------|
| 日 | 28px | 年月 | 日付（土日赤・Today青） |
| 週 | 56px | 年月 | 週開始日（月曜） |
| 月 | 112px | 年 | 月 |
| 四半期 | 224px | **年度**（4月始まり） | Q1〜Q4 |
| 年 | 336px | — | 年 |

スケール切替ボタンとイナズマ線ボタンはガントビュー内の専用サブバーに配置。タブ切替時・スケール切替時は Today が画面横中央に自動スクロール。

#### 3.2 四半期の定義（4月始まり年度）

| 表示 | 期間 |
|------|------|
| Q1 | 4月〜6月 |
| Q2 | 7月〜9月 |
| Q3 | 10月〜12月 |
| Q4 | 翌年1月〜3月 |

#### 3.3 レイアウト固定

| 固定方式 | 対象 |
|---------|------|
| 左固定（sticky left） | タスク名列（横スクロール時も固定） |
| 上固定（sticky top） | ヘッダー行（縦スクロール時も固定） |
| 両方固定 | 交差点セル |

単一スクロールコンテナ（`overflow:auto`）構造で縦横両方向スクロールと sticky 固定を両立。

#### 3.4 バーの精度

`ganttScaleDateToX(date)` が各スケールの列幅に対して日数比率で正確なピクセル位置を返す。`taskLeft` = 開始日の左端、`taskRight` = 終了日翌日の左端。

#### 3.5 バー操作（ドラッグ＆ドロップ）

| 操作 | 動作 |
|------|------|
| バー中央をドラッグ | 開始日・終了日を同時にシフト |
| バー左端をドラッグ | 開始日のみ変更 |
| バー右端をドラッグ | 終了日のみ変更 |

ドラッグ中はツールチップに日付と日数をリアルタイム表示。

#### 3.6 依存関係の矢印

| 種別 | 表示 |
|------|------|
| 正常 | グレー点線矢印 |
| 期間重複 | 赤点線矢印 + ツールチップ警告 |

#### 3.7 今日ライン・タスク選択スクロール

- 青い縦線で今日の日付を表示。「TODAY」バッジ付き
- タスク名クリック時：開始日が画面左1/4に来るよう自動スクロール

#### 3.8 イナズマ線

全スケールで `ganttScaleDateToX` を使用し、スケール切替後も正しく追従。

| マーカー | 意味 |
|---------|------|
| ◆ 青（始点・終点） | Today ライン上のアンカー点 |
| ● 青 | 今日ラインより右（進んでいる） |
| ● 赤 | 今日ラインより左（遅れている） |
| ◆ 赤 | 期間超過かつ未完了の遅延タスク |

### 7. ストレージ管理

#### 7.1 ストレージモード

| モード | 保存先 | 特徴 |
|--------|--------|------|
| ローカルストレージ | ブラウザ内蔵 | 設定不要。単一 PC のみ |
| OneDrive | 指定フォルダの `wbs_data.json` | チーム共有可能 |

#### 7.2 起動ウィザード（Step1で完結する設計）

```
起動
└─ Step1：ストレージ選択
    ├─ [前回と同じ設定で続行] → 自動ログイン（前回ユーザーあり時）
    ├─ [⚡ ワンクリックで再接続] → OneDrive接続 → 自動ログイン
    ├─ [💾 ローカルストレージ] → 自動ログイン（または Step2へ）
    └─ [☁️ OneDrive] → フォルダ選択ダイアログ → 自動ログイン（または Step2へ）

Step2（前回ユーザーなし・初回のみ）：ユーザー選択
```

#### 7.3 OneDrive モードの動作

| ファイル | 用途 |
|---------|------|
| `wbs_data.json` | 全プロジェクト・ユーザーデータ |
| `wbs_lock.json` | 編集中ユーザー情報（`locked` / `lockedBy` / `userId` / `expires` / `clearedAt`） |

#### 7.4 参照モードの制御

参照モード中は以下すべてが無効化される：

- タスクの追加・編集・削除・ドラッグ
- 詳細パネルの全入力フィールド（input / select / textarea / range）
- プロジェクトの追加・削除
- ガントチャートのバードラッグ
- localStorage への書き込み（タイムスタンプ更新防止）

`updateEditModeUI` の判定は `storageMode==='csvFile'` のみ（`csvDirHandle` の有無に依存しない）。

#### 7.5 排他制御の設計

- ロックファイルの書き込みは `writeLockFile(true/false)` に統一
- `saveToCsvDir` は編集モード中のみ `writeLockFile(true)` を呼ぶ
- `checkLockFile` は `locked:false` / `userId:null` / 有効期限切れ / 自分のロック をすべて null 扱い
- チェックアウト時は `locked:false` + `clearedAt` タイムスタンプを明示的に書き込む

#### 7.6 フォルダハンドル永続化

| 保存先 | 内容 | 用途 |
|--------|------|------|
| IndexedDB（`onedrive-folder`） | `FileSystemDirectoryHandle` | ワンクリック再接続 |
| localStorage（`wbs_storage_mode_v1`） | フォルダ名文字列 | UI表示ヒント |

### 8. CSV インポート / エクスポート

#### 8.1 エクスポート形式

```
ID, No, タスク名, 種別, 成果物, ステータス, 担当者,
開始日, 終了日, 進捗率, 依存関係（ID）, 依存関係（No）, 親ID, 備考
```

エクスポートボタン：上向き矢印SVGアイコン。インポートボタン：下向き矢印SVGアイコン。

#### 8.2 インポート動作

| 条件 | 動作 |
|------|------|
| ID 列あり + 既存データあり | アップサート（更新）または上書きを選択 |
| ID 列なし（旧形式） | 全体を上書き |
| 日付フォーマット | YYYY-MM-DD 以外も Date オブジェクト経由で正規化 |

## データモデル

### プロジェクト

```json
{
  "id": "p1",
  "name": "プロジェクト名",
  "desc": "説明",
  "color": "#2563EB",
  "tasks": []
}
```

### タスク

```json
{
  "id": "t12",
  "parentId": "t11",
  "type": "task | folder",
  "name": "タスク名",
  "deliverable": "成果物定義",
  "status": "todo | doing | done | block",
  "assignee": "担当者名",
  "start": "2026-05-12",
  "end": "2026-06-06",
  "progress": 55,
  "deps": ["t7", "t8"],
  "note": "備考",
  "hidden": false
}
```

### ユーザー

```json
{
  "id": "u1",
  "name": "田中 太郎",
  "role": "プロジェクトマネージャー",
  "note": "",
  "color": "#2563EB",
  "createdAt": "2026-01-01T00:00:00.000Z"
}
```

### ロックファイル（wbs_lock.json）

```json
{
  "locked": true,
  "lockedBy": "田中 太郎",
  "userId": "u1",
  "lockedAt": "2026-06-04T10:00:00.000Z",
  "expires": "2026-06-04T12:00:00.000Z"
}
```

チェックアウト時：`locked:false`、全フィールドをnullクリア、`clearedAt` タイムスタンプを追記。
