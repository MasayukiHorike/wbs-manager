# WBS Manager 設計書

## 概要

| 項目 | 内容 |
|------|------|
| ツール名 | WBS Manager |
| バージョン | v1.34.0 |
| 形態 | シングル HTML ファイル（バックエンドなし） |
| 動作環境 | Chrome / Edge（File System Access API 対応ブラウザ） |
| データ保存 | localStorage / OneDrive 同期フォルダ（JSON） |
| 開発言語 | HTML / CSS / Vanilla JavaScript（約5,900行） |

## 画面構成

```
ヘッダー
  ロゴ | [📋リスト] [📊ガントチャート] [🏠ダッシュボード] [📝改善要望]

  プロジェクト一覧・追加・削除・統計
```

## 機能一覧

### 1. タスク・フォルダ・マイルストーン管理

#### 1.1 種別

| 種別 | アイコン | 概要 |
|------|---------|------|
| フォルダ | 📁📂 | タスクをグループ化。進捗率は子孫タスクの期間加重平均で自動計算 |
| タスク | 📄 | 実作業単位。進捗率は手動入力 |
| マイルストーン | 🔷 | 節目日。`start` のみ使用（終了日なし）。ガントで◆ひし形マーカー表示 |

#### 1.2 タスク属性

| 属性 | 概要 |
|------|------|
| 名前 | タスク・フォルダ・マイルストーンの名称 |
| 成果物定義 | 完了条件・成果物 |
| ステータス | 未着手 / 進行中 / 完了 / ブロック中 |
| 担当者 | `<input list>` + `<datalist>` の1ボックス（登録ユーザー候補＋自由入力） |
| 開始日 / 終了日 | YYYY-MM-DD形式。子タスク追加時は親の日程を継承 |
| 進捗率 | タスクのみ手動入力。フォルダは自動計算 |
| 依存関係 | 前工程タスクのIDを複数指定 |
| 備考 | メモ・補足情報 |
| 非表示フラグ | `hidden:true` でリスト・ガントから非表示（データは保持） |

#### 1.3 CRUD 操作

| 操作 | 方法 |
|------|------|
| 追加 | ヘッダーボタン / 行左端のSVGアイコン / 右クリックメニュー |
| 挿入位置 | クリックした行のサブツリー直後に正確に挿入（`getSubtreeLastIndex`） |
| 編集 | 行クリックで右パネル表示、フィールド直接編集 |
| 削除 | 行左端の🗑アイコン（子孫も同時削除） |
| 並替 | 「⠿ 並替モード」ON後、ドラッグ＆ドロップ |
| 非表示 | 行左端の👁アイコン（個別トグル） |

### 2. リストビュー

| 機能 | 概要 |
|------|------|
| 階層インデント表示 | 深さに応じたインデント・折りたたみ |
| 行番号（No） | 動的計算（例：1, 1.1, 2.3.1） |

| ヘッダー固定 | `padding-top:0` でスクロール時も最上端に固定 |
| タイプバッジ | TASK（グレー）/ FOLDER（青）/ MILESTONE（紫） |

### 3. ガントチャート

#### 3.1 表示スケール・ツールバー

ツールバーボタン順（左から）：①表示レンジ ②イナズマ線 ③クリティカルパス ④色分け ⑤印刷/PDF

| スケール | 列幅 | 四半期定義 |
|---------|------|-----------|
| 日 | 28px | — |
| 週 | 56px | — |
| 月 | 112px | — |
| 四半期 | 224px | **4月始まり年度**（Q1=4〜6月、Q2=7〜9月、Q3=10〜12月、Q4=翌年1〜3月） |
| 年 | 336px | — |

#### 3.2 バー色分けルール

| ルール | 内容 |
|--------|------|
| ステータス | 未着手=グレー / 進行中=青 / 完了=緑 / ブロック=赤 |
| 担当者 | 各ユーザーのアバターカラー |
| 遅延状況 | 完了=緑 / 期限超過=赤 / 遅れ気味=オレンジ / 順調=青 |
| なし | デフォルト（ステータスと同じ） |

#### 3.3 マイルストーン表示

- ガントチャート：◆ひし形マーカー（ステータス連動カラー）＋タスク名ラベル
- `end` が null でも `start` のみで正しく描画
- フォルダ進捗計算・イナズマ線の対象外

#### 3.4 クリティカルパス

CPM法（前向きES/EF・後向きLS/LF・フロート=0判定）で計算。

- バーに赤アウトライン＋赤シャドウ
- ラベル列に赤左ボーダー＋薄赤背景
- ガントツールバーの「🔴 クリティカルパス」ボタンでON/OFF

#### 3.5 その他機能

| 機能 | 概要 |
|------|------|
| バーD&D | 移動（日程シフト）・左右リサイズ |
| 依存矢印 | 正常=グレー点線 / 重複=赤点線 |
| イナズマ線 | 全スケールで `ganttScaleDateToX` に追従 |
| 担当者アバター | ラベル列のタスク行に18px丸アバター表示 |
| タスク選択スクロール | クリック時に開始日が画面左1/4に自動スクロール |
| フィルタ連動 | リストビューのフィルタがガント行にも適用 |
| 印刷/PDF | A3横向きで印刷プレビュー |

### 4. ダッシュボード（🏠タブ）

#### 4.1 カード一覧（13種類）

| カード | 内容 |
|--------|------|
| KPI | 総タスク数・完了・期限超過・ブロック中 |
| 🏥 健全性スコア | 0〜100点ゲージ（進捗遅延・超過・ブロックで減点） |
| 🍩 ステータス分布 | SVGドーナツチャート |
| 📉 バーンダウン | 理想ライン・実績ライン・今日線。カード幅に追従するレスポンシブSVG |
| 📁 プロジェクト別完了率 | カラーバー＋期限超過件数 |
| 👤 担当者別タスク数 | 完了/未完了の積み上げ横棒 |
| ⚠ 遅延リスクタスク | 計画-実績≥10%のタスクを降順一覧 |
| 📅 今週の作業予定 | 今日〜7日以内の開始・終了・進行中タスク |
| ⚖ ワークロード分析 | 担当者ごとの負荷（進行中・今週締切・超過）と負荷レベル判定 |
| 🕐 更新履歴 | Undoログから最新15件を表示 |
| 🔴 クリティカルパス | ダッシュボード独立CPM計算によるCPタスク一覧 |
| 📊 フェーズタイムライン | 最上位フォルダを横棒タイムラインで表示 |
| 🔷 マイルストーン | 直近7日以内のマイルストーン |

#### 4.2 カード設定（ユーザーごとに保存）

- **表示/非表示**：各カードを個別ON/OFF
- **幅**：標準（1列）/ 2列 をボタンで切替
- **並べ替え**：設定モーダル内で ⠿ ドラッグにより順序変更
- **保存先**：`wbs_db_prefs_v2`（localStorage）
- **リセット**：全カードをデフォルト設定に戻す

### 7. ストレージ管理

#### 7.1 起動ウィザード（Step1で完結する設計）

```
起動 → Step1：ストレージ選択
  ├─ [前回と同じ設定で続行] → 自動ログイン
  ├─ [⚡ ワンクリックで再接続] → OneDrive接続 → 自動ログイン
  ├─ [💾 ローカルストレージ] → 自動ログイン
  └─ [☁️ OneDrive] → フォルダ選択 → 自動ログイン

Step2（前回ユーザーなし・初回のみ）：ユーザー選択
```

前回ログインしたユーザーが有効な場合はStep2をスキップして自動ログイン。

#### 7.2 OneDriveモードのファイル構成

| ファイル | 用途 |
|---------|------|
| `wbs_data.json` | 全プロジェクト・ユーザーデータ |
| `wbs_lock.json` | 排他制御（locked / lockedBy / userId / expires / clearedAt） |
| `wbs_feedback.json` | 改善要望データ |

#### 7.3 参照モードの制御

参照モード中は `isReadOnly()` / `guardReadOnly()` で以下をブロック：

- タスク追加・編集・削除・並べ替え
- ガントバードラッグ・依存関係操作
- プロジェクト追加・削除
- CSVインポート
- 詳細パネルの全入力フィールド（CSS pointer-events:none + JSガード節の二重保護）

#### 7.4 IDカウンタ管理

- `_uid` 初期値：200（サンプルデータt100〜t116との衝突防止）
- `resyncUid()`：データ読み込み後に既存タスクIDをスキャンして安全な値に補正

### 8. セキュリティ

| 対策 | 内容 |
|------|------|
| XSS防止 | `escHtml()` で `&`・`<`・`>`・`"`・`'` をエスケープ。全innerHTML出力箇所に適用 |
| データバリデーション | `validateWbsData()` で読み込みデータのスキーマ検証 |
| 参照モードガード | UIのCSS制御＋関数レベルのガード節で二重保護 |

### 9. CSV インポート / エクスポート

```
ID, No, タスク名, 種別, 成果物, ステータス, 担当者,
開始日, 終了日, 進捗率, 依存関係（ID）, 依存関係（No）, 親ID, 備考
```

| 条件 | 動作 |
|------|------|
| ID列あり | アップサート（更新）または上書きを選択 |
| ID列なし | 全体を上書き |

## データモデル

### タスク

```json
{
  "id": "t200",
  "parentId": "t116",
  "type": "task | folder | milestone",
  "name": "タスク名",
  "deliverable": "成果物定義",
  "status": "todo | doing | done | block",
  "assignee": "担当者名",
  "start": "2026-05-12",
  "end": "2026-06-06",
  "progress": 55,
  "deps": ["t7", "t8"],
  "note": "備考",
  "hidden": false
}
```

### ロックファイル（wbs_lock.json）

```json
{
  "locked": true,
  "lockedBy": "田中 太郎",
  "userId": "u1",
  "lockedAt": "2026-06-04T10:00:00.000Z",
  "expires": "2026-06-04T12:00:00.000Z"
}
```

チェックアウト時：`locked:false`、全フィールドをnullクリア、`clearedAt` タイムスタンプを追記。
