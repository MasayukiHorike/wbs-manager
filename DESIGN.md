## 機能一覧

### 1. タスク・フォルダ・マイルストーン管理

#### 1.1 種別

| 種別 | アイコン/バッジ | 概要 |
|------|----------------|------|
| フォルダ | 📁📂 / FOLDER（青） | グループ化。進捗率は子タスクの期間加重平均で自動計算 |
| タスク | 📄 / TASK（グレー） | 実作業単位。進捗率は手動入力 |
| マイルストーン | 🔷 / MILESTONE（紫） | 節目日。`start` のみ使用。ガントで◆ひし形マーカー |

#### 1.2 タスク属性

| 属性 | 概要 |
|------|------|
| 名前 | タスク名 |
| 成果物定義 | 完了条件・成果物 |
| ステータス | 未着手 / 進行中 / 完了 / ブロック中 |
| 担当者 | 登録ユーザー候補＋自由入力の1ボックス |
| 開始日 / 終了日 | YYYY-MM-DD。子追加時は親日程を継承 |
| 進捗率 | タスクのみ手動。フォルダは自動計算。マイルストーンは完了=100%/それ以外=0% |
| 依存関係 | 前工程タスクのIDを複数指定 |
| 備考 | メモ・補足 |
| 非表示フラグ | `hidden:true` でリスト・ガントから非表示（データ保持） |

#### 1.3 CRUD 操作

| 操作 | 方法 |
|------|------|
| 追加 | ヘッダー左側ボタン / 行左端アイコン / 右クリックメニュー |
| 挿入位置 | クリックした行のサブツリー直後（`getSubtreeLastIndex`） |
| 編集 | 行クリック → 右パネル |
| 削除 | 行左端🗑アイコン（子孫も同時削除） |
| 並替 | 並替モードON後、ドラッグ。自身の子孫への移動は3層バリデーションでブロック |
| 非表示 | 行左端👁アイコン |

### 3. ガントチャート

#### 3.1 表示スケール

| スケール | 列幅 | 1画面の目安 |
|---------|------|------------|
| 日 | 40px | 約30日 |
| 週 | 90px | 約3ヶ月 |
| 月 | 100px | 約1年 |
| 四半期（4月始まり） | 200px | 約1.5年 |
| 年 | 400px | 約3年 |

#### 3.2 ツールバー（左から）

① 表示レンジ ② ⚡イナズマ線 ③ 🔴クリティカルパス ④ 色分けルール ⑤ 🖨印刷/PDF

#### 3.3 バー色分けルール

| ルール | 内容 |
|--------|------|
| ステータス | 未着手=グレー / 進行中=青 / 完了=緑 / ブロック=赤 |
| 担当者 | 各ユーザーのアバターカラー |
| 遅延状況 | 完了=緑 / 超過=赤 / 遅れ=オレンジ / 順調=青 |
| なし | デフォルト（ステータスと同じ） |

#### 3.4 マイルストーン

- ガント：◆ひし形マーカー（ステータス連動カラー）＋タスク名ラベル
- `end` が null でも `start` のみで正しく描画
- フォルダ進捗計算・イナズマ線の対象外

#### 3.5 クリティカルパス

CPM法（前向きES/EF・後向きLS/LF・フロート=0判定）で自動計算。バーに赤アウトライン、ラベル列に赤左ボーダー。

#### 3.6 ラベル列幅リサイズ

ガントチャートのラベル列（タスク名列）右端をドラッグして列幅を変更できます。

| 項目 | 内容 |
|------|------|
| デフォルト幅 | 320px |
| 調整範囲 | 160px〜600px |
| ハンドル | ラベルヘッダー右端の透明8px帯（カーソルが `col-resize` に変化） |
| 確定タイミング | マウスアップ後にガントを再描画 |

#### 3.7 ヘッダー・ラベル列の固定（Sticky）

| スクロール方向 | 固定される要素 | 実装方式 |
|-------------|-------------|---------|
| 縦スクロール | 日付ヘッダー行（年月・日付） | ヘッダー行に `position:sticky; top:0; z-index:30` |
| 横スクロール | タスク名列・タスク名ヘッダー交差点 | `position:sticky; left:0` を外側ラッパーに適用。内側で `position:relative` を持つ子要素でリサイズハンドル等を管理 |

**設計上の注意：** `position:sticky` と `position:relative/absolute` は同一要素に共存不可（後者が優先される）。必ず外側/内側のラッパー分離が必要。

#### 3.8 フィルタ連動

サイドバーの全フィルタがガントチャートの表示行にも連動。

### 7. ストレージ管理

#### 7.1 起動ウィザード

起動時に表示。前回設定がある場合は状況に応じた続行ボタン1つを表示（2ボタン問題を解消）。

| 続行ボタンの表示 | 条件 |
|----------------|------|
| ⚡ OneDriveに接続して続行（緑） | 前回OneDrive＋フォルダ記憶あり |
| ☁️ OneDriveモードで続行 | 前回OneDrive＋フォルダ未記憶 |
| 💾 ローカルストレージで続行 | 前回ローカル |
| （なし） | 初回起動 |

前回ログインユーザーが有効な場合はユーザー選択画面をスキップして自動ログイン。

#### 7.2 OneDriveモードのファイル構成

| ファイル | 用途 |
|---------|------|
| `wbs_data.json` | 全プロジェクト・ユーザーデータ |
| `wbs_lock.json` | 排他制御（locked / lockedBy / userId / expires / clearedAt） |
| `wbs_feedback.json` | 改善要望データ |
| `Backup/YYYY-MM-DD_HHmmss/` | 自動バックアップ（編集開始時に作成・最大10世代） |

#### 7.3 ヘッダーの状態表示（OneDriveモード）

| 表示 | 状態 |
|------|------|
| 👁 参照モード（黄バナー） | 起動直後・参照中 |
| 🔒 ○○が編集中（赤バナー） | 他ユーザー編集中 |
| ✏️ 編集中（緑バッジ） | 自分が編集モード |

#### 7.4 参照モードのガード

UI（CSS `pointer-events:none`）＋関数レベル（`guardReadOnly()`）の二重保護。対象：タスク追加・編集・削除・並替・ガントバードラッグ・依存関係操作・CSVインポート・プロジェクト追加・削除。

#### 7.5 IDカウンタ管理

- `_uid` 初期値：200（サンプルデータ t100〜t116 との衝突防止）
- `resyncUid()`：以下のタイミングで既存IDをスキャンして安全な値に補正
  - データ読み込み（`applyLoadedData`）後
  - CSVインポート後
  - プロジェクト切替（`switchProject`）後

#### 7.6 自動バックアップ（OneDriveモードのみ）

「✏️ 編集開始」を押すたびに実行。失敗しても編集モード移行はブロックしない。

| 項目 | 内容 |
|------|------|
| 保存先 | `csvDirHandle/Backup/タイムスタンプ/`（フォルダは自動作成） |
| タイムスタンプ形式 | `YYYY-MM-DD_HHmmss`（例：`2026-06-08_143022`） |
| 保存ファイル | `wbs_data.json` + `wbs_feedback.json` |
| 世代管理 | 最大10件。超過分は古い順に `removeEntry({recursive:true})` で自動削除 |
| 失敗時 | トースト警告を表示し、編集モードは通常通り開始 |

復元方法：エクスプローラーで `Backup/` フォルダを開き、復元したい世代の `wbs_data.json` をルートにコピーして上書き。

### 9. CSV インポート / エクスポート

```
ID, No, タスク名, 種別, 成果物, ステータス, 担当者,
開始日, 終了日, 進捗率, 依存関係（ID）, 依存関係（No）, 親ID, 備考
```

**種別の値：** `task` / `folder` / `milestone`（英語表記・大文字小文字区別なし）

**ステータスの値：** `todo` / `doing` / `done` / `block`（英語表記・大文字小文字区別なし・日本語も吸収）

| 条件 | 動作 |
|------|------|
| ID列あり | アップサート（更新）または上書きを選択 |
| ID列なし | 全体を上書き |
| 大文字小文字混在 | `parseType()`・`parseStatus()` が正規化して吸収 |
| インポート完了後 | `resyncUid()` + `saveToLS()` を明示的に実行 |
