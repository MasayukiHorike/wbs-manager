# CHANGELOG — WBS Manager

開発の経緯と各バージョンの変更内容を記録します。

---

## v1.0.0 — 初回リリース

**基本的なWBS管理ツールとして実装**

- タスクの階層構造（インデント）表示
- 担当者・期日の管理
- 進捗率・ステータス管理（未着手 / 進行中 / 完了 / ブロック中）
- ガントチャート表示（日単位）
- テキスト検索・ステータス・担当者フィルタ
- CSVエクスポート（BOM付きUTF-8）
- ライトテーマ（クリーン）デザイン

## v1.2.0 — ガントチャート強化・タスクNo・CSVインポート

## v1.23.0 — ストレージ・排他制御の安定化

- **起動時ウィザード**：ストレージ選択（ローカル / OneDrive）とユーザー選択を1画面に統合（Step1→Step2のステップ式）
- **参照モードの完全ガード**：CSVモード起動時は `storageMode` の確定タイミングを修正し、フォルダ未接続でも参照モードを正しく維持
- **詳細パネルの参照モードロック**：入力フィールド全種（input / select / textarea / range）を `pointer-events:none` でロック。パネル上部に「👁 参照モード」バナーを表示
- **プロジェクト追加・削除の参照モードガード**：参照モード時はプロジェクト操作をブロックしトースト通知を表示
- **排他制御バグ修正**：`saveToCsvDir` が独自フォーマットでロックファイルを上書きしていた問題を修正。`writeLockFile` に統一し `locked` フィールドを必ず含める設計に変更
- **チェックアウト後のロック残存バグ修正**：`checkLockFile` に `!lock.userId` ガードを追加。`locked:false` 時の `clearedAt` タイムスタンプを追記
- **OneDriveフォルダハンドル永続化**：IndexedDB に `FileSystemDirectoryHandle` を保存しワンクリック再接続を実現
- **localStorageへの誤書き込み防止**：参照モード中は localStorage へも書き込まないよう `saveToLS` を修正
